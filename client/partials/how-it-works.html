How it works

Icecomm abstracts away the complexity of setting up WebRTC for developers. In order for a peer-to-peer connection to happen between browsers, a framework known as Interactive Connectivity Establishment (ICE) must be used to exchange the information neccessary for the browsers to connect directly.

MDN WebRTC Overview

With WebRTC, the ICE candidates exchange information via STUN (Session Traversal Utilities for NAT [Network Address Translation]) and TURN (Traversal Using Relays around NAT) servers. SDP (Session Description Protocol) information is passed between peers, which translates the data into a standard readable by all connected peers. For developers using Icecomm, this server communication is all handled behind the scenes.
